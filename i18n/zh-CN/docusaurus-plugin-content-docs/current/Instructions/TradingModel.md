---
title: ExinOne 交易业务模式 
---



ExinOne 聚合市面上交易深度好的交易所为 Mixin 内的用户提供交易服务，比如：Binance、OKX、Gateio 等，使您能在 ExinOne 上享受跟交易所一致的交易深度，现在为您介绍 ExinOne 交易的业务模式：



## 名词引入

- 资金池：为了提供交易，ExinOne 需有一定代币储备供您买/卖（货）
- 链上充提：指发生在交易所和 Mixin 内的充提币，由于区块链充提币需要时间确认，

- 延迟放币：部分情况下（下面会介绍），由于无法将币立即转入到用户 Mixin 钱包，需要延迟一会儿转入，即延迟放币
- 流动性：指 ExinOne 的资金池支撑用户交易的情况

## 案例说明

以 BTC、ETH、USDT 为例：

ExinOne 需在交易所和 Mixin 内分别存放一定数量的 BTC、ETH、USDT，假设分别是 100 个 BTC，1000 个 ETH、1000000 个 USDT

所以初始资金池为：

| 代币 | ExinOne | 交易所 |
| ---- | ------- | ------ |
| BTC  | 50      | 50     |
| ETH  | 100     | 100    |
| USDT | 500000  | 500000 |

有了这些资产后，就可以提供交易了，假设现在有用户支付 30000 USDT 给 ExinOne，购买 1 个 BTC，ExinOne 的资产变化为：

| 代币 | ExinOne | 交易所 |
| ---- | ------- | ------ |
| BTC  | 49      | 51     |
| ETH  | 100     | 100    |
| USDT | 530000  | 470000 |

可以看到，BTC 和 USDT 在 ExinOne 和交易所上的数量有了变化：

ExinOne 里的 USDT 增加了（因为用户支付了 30000 USDT 给 ExinOne），BTC 减少了（因为 ExinOne 把 1 个 BTC 转给了 用户）

同时，交易所的 USDT 和 BTC 跟 ExinOne 的情况相反 —— ExinOne 帮用户在交易所买了币。



假设当前币价波动，暴跌，用户开始“抄底”，则 ExinOne 的 USDT 会增加，购买 BTC、ETH 等代币，假设此时用户支付 40 万 USDT 购买 BTC 和 ETH，购买完成后，如下图，ExinOne 的 USDT 将达到 93 万枚，而交易所的 USDT 将只有 7 万枚

| 代币 | ExinOne | 交易所 |
| ---- | ------- | ------ |
| BTC  | -       | -      |
| ETH  | -       | -      |
| USDT | 930000  | 70000  |

意味着用户再购买 7 万枚就无法购买了，因为交易所的 USDT 被消耗没了。

此时即需要「平衡资金池」：将 ExinOne 上的一些 USDT 提至交易所，从而维持资金池的稳定。

如果用户大量购买 BTC，导致 ExinOne 内的 BTC 被消耗尽，则 ExinOne 暂时无法将币转账到用户的 Mixin 钱包，此时则会将 BTC 从交易所提币到 Mixin 内以放币，但由于链上转账需要时间，所以用户可能会看到“延迟放币”，不同币的到账时间不同，可能在几分钟到几个小时之间。

## 延伸

根据以上业务模式的介绍，请思考：为什么有些币只能入账到交易账户？

以下是解答：

如前文所说，一切代币能够提供交易，都需要有流动性做支撑，按以往的业务模式下，需要在 Mixin 内和交易所上同时构建资金池，否则就需要频繁提币来维持资金池的平衡，这里有些问题：

1. 提币成本极高
2. 单币如果交易量不足，无法覆盖提币成本
3. 如果 Mixin 不支持某条公链，ExinOne 将无法提供相关代币的交易服务

这样的结果导致 ExinOne 无法支持更多交易对，为了解决这个问题，ExinOne 将交易账户内的部分币设置为“仅入账到交易账户”，这样，用户购买的币全都存在对应的交易所内，ExinOne 并不会将相关代币用作它途，故用户可以随时进行自由买卖。

由此，ExinOne 能够支持的交易对也就更多了。

其次，部分代币由于链上确认时间极长，比如 ETC，大约需要 40 小时，在极端行情下，某一方资金池消耗空，补充时间可能会很久，这期间可能无法交易了，所以限制入账交易账户可以解决这个问题。
